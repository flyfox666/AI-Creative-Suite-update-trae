## 目标
- 不再在视频预览区域旁边显示“Files API 上传信息”卡片，同时保留后端日志与必要的上传流程，以免打扰用户界面。

## 修改点
- `components/MediaAnalyzer.tsx`
  - 移除或条件隐藏上传信息卡片的渲染块（当前固定在“分析视频”按钮上方右侧）。
  - 保留上传流程与错误处理；如需内部调试，继续在终端查看日志代理输出，不在 UI 展示。
  - 可选：保留 `videoUploadInfo` 状态仅用于内部判断，但不做渲染。

## 保留与调试
- 日志代理终端仍可实时看到上传阶段与 `file_uri`，用于排查问题（`npm run proxy`）。
- 设置页“Gemini 文件上传健康检查”按钮仍可独立验证上传连通性。

## 验证
- 上传视频后界面不展示上传信息卡片；点击“分析视频”正常分析。
- 终端可通过日志代理看到实时请求/响应，用于定位潜在错误。
