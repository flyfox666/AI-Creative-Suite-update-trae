## 问题原因
- 当前 `contexts/LocalizationContext.tsx` 初始语言为 `'en'`，导致首页首次加载显示英文。

## 修改方案
1) 默认语言改为中文优先
- 在 `LocalizationContext` 初始化时改为：
  - 先读 `localStorage('ui.locale')`，如果是 `'zh'|'en'` 则使用；
  - 否则检查 `navigator.language` 是否以 `zh` 开头，是则使用 `'zh'`；
  - 最终回退默认 `'zh'`。
- 代码示例：
  ```ts
  const getInitialLocale = (): Locale => {
    try {
      const saved = window.localStorage.getItem('ui.locale')
      if (saved === 'zh' || saved === 'en') return saved as Locale
      const nav = (navigator.language || (navigator.languages?.[0] ?? '')).toLowerCase()
      if (nav.startsWith('zh')) return 'zh'
    } catch {}
    return 'zh'
  }
  const [locale, setLocale] = useState<Locale>(getInitialLocale())
  ```

2) 切换语言时持久化
- 在 `setLocale` 时同步写入 `localStorage('ui.locale', value)`，刷新后保留选择：
  ```ts
  const changeLocale = (next: Locale) => {
    setLocale(next)
    try { window.localStorage.setItem('ui.locale', next) } catch {}
  }
  ```
- 保持 `LanguageSwitcher` 使用 `changeLocale`（或在其内部写入 localStorage）。

3) 资源加载保持不变
- 继续使用 `${import.meta.env.BASE_URL}locales/en.json` 与 `zh.json` 加载，不需改动。

## 验证
- 清空浏览器 `localStorage('ui.locale')` 后打开首页，默认显示中文。
- 切换到英文后刷新，仍保持英文。

## 影响范围
- 仅影响 UI 文案默认语言；不更改“AI 回复语言”（`settings.REPLY_LANGUAGE` 仍按现有逻辑）。
